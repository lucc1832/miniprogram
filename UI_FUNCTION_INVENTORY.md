# 小小工具箱：现有功能与 UI 设计清单

更新时间：2026-06-06

## 1. 产品定位

这是一个微信生活工具小程序，把日常记录与轻量管理能力集中在一个入口中。

当前主要模块：

1. 到期便签与使用成本
2. 天气预报
3. 纪念日
4. 我的厨房
5. 体重管理
6. 月经记录
7. 个人中心与云端数据管理

记账模块目前仅有首页占位入口，尚未开发。

## 2. 全局导航

### 底部导航

- 首页：展示所有模块与最近使用模块。
- 我的：个人资料、账号、云端同步、偏好设置和支持信息。

### 首页

- 展示当天日期。
- 展示当前已启用模块数量。
- 展示最近一次打开的模块，并支持一键继续。
- 模块卡片进入对应功能。
- 对尚未开放的模块显示“即将上线”状态。
- 防止快速重复点击导致重复跳转。

## 4. 到期便签与使用成本

### 4.1 模块作用

同一套物品数据分为两种管理模式：

- 到期便签：管理食品、药品、证件等物品的有效期。
- 使用成本：计算物品购买后的日均成本与回本进度。

### 4.2 首页功能

- 在“到期提醒”和“使用成本”之间切换。
- 按名称或分类搜索。
- 展示统计摘要。
- 展示今日最需要关注的物品。
- 点击统计卡片快速筛选。
- 按分类筛选。
- 按状态筛选。
- 切换排序方式。
- 查看物品列表。
- 点击物品进入详情。
- 浮动按钮新增物品。
- 空数据时展示引导状态。

### 4.3 到期提醒模式

- 展示总资产、全部物品数、30 天内到期数量。
- 自动识别长期、未到期、临期、已过期状态。
- 展示剩余天数或已超期天数。
- 支持按到期日、购买日排序。
- 支持全部、未过期、临期、已过期筛选。

### 4.4 使用成本模式

- 展示总投入、今日摊销、最高日成本与对应物品。
- 计算使用天数。
- 计算日均成本。
- 计算预计回本日与回本进度。
- 支持使用中、已归档筛选。
- 支持按日成本、价格排序。
- 自动生成每日回本记录。

### 4.5 新增与编辑物品

- 切换记录类型：到期便签或使用成本。
- 输入物品名称。
- 选择已有分类。
- 在表单内直接新增分类。
- 选择购买日期。
- 到期便签填写到期日期。
- 使用成本填写购买价格。
- 设置使用中或归档状态。
- 保存新增物品。
- 编辑已有物品。
- 删除已有物品。

### 4.6 截图 OCR

- 从相册或相机选择截图。
- 上传图片到云存储。
- 调用百度 OCR 云函数。
- 尝试识别商品名、购买日期和价格。
- 自动填入识别成功的字段。
- 识别失败时展示可理解的错误提示，并允许手动填写。

### 4.7 分类管理

- 新增分类。
- 删除分类。
- 分类上移、下移排序。
- 空分类状态。

### 4.8 物品详情

- 展示物品类型、分类和状态。
- 展示购买日期、到期日期和购买价格。
- 展示使用天数和日均成本。
- 展示到期状态或回本进度。
- 进入编辑页面。
- 使用成本物品支持归档与恢复使用。

### 4.9 数据备份页面

- 导出分类、物品和记录为 JSON，并复制到剪贴板。
- 粘贴 JSON 导入数据。
- 当前导入方式会新增数据，不自动去重。
- 当前页面已注册，但主流程没有明显入口。

## 5. 天气预报

### 5.1 模块作用

基于高德地图天气 API 展示当前城市实时天气和未来趋势，并管理多个城市。

### 5.2 天气主页

- 获取当前定位城市。
- 定位失败时使用北京作为默认城市。
- 展示实时温度、天气、最高温、最低温。
- 展示风向、风力、湿度、体感、数据源和发布时间。
- 展示穿衣建议。
- 展示根据天气生成的今日洞察。
- 展示白天和夜间趋势。
- 展示未来天气预报与温度区间。
- 展示趋势、降水和温度摘要。
- 根据天气和昼夜切换背景主题。
- 下拉刷新天气。
- 左右滑动切换城市。
- 点击城市名打开城市切换面板。

### 5.3 天气缓存与性能逻辑

- 每个城市独立缓存天气数据。
- 普通缓存有效时间为 2 小时。
- 手动强制刷新在 10 分钟内会使用最近数据，避免频繁请求。
- 同一个城市的并发请求会合并。
- 定位城市每小时最多静默更新一次。
- API 失败时优先展示旧缓存。

### 5.4 城市管理

- 展示已添加城市及简要天气。
- 点击城市切换。
- 编辑模式批量选择城市。
- 全选城市。
- 批量删除。
- 左滑单独删除。
- 当前定位城市不可删除。
- 添加城市。

### 5.5 搜索城市

- 按城市名称搜索。
- 输入搜索使用延迟请求，避免频繁调用 API。
- 显示搜索结果。
- 快速添加当前位置。
- 快速添加热门城市与更多城市。
- 已存在城市直接切换，不重复添加。

### 5.6 天气图标主题

- 支持默认 Emoji 和扁平示例主题。
- 自定义图标上传目前只是演示功能。
- 图标管理页面文件存在，但当前未注册到 `app.json`，从天气菜单进入可能无法正常打开。

## 6. 纪念日

### 6.1 模块作用

记录已经发生的重要纪念日，以及未来需要倒数的日期。

### 6.2 纪念日首页

- 展示每日情绪文案。
- 自动挑选一个最值得关注的主事件。
- 展示次要关注事件。
- 按全部、倒数、今天、重要等维度展示统计。
- 展示今天发生的事件。
- 展示即将到来的倒数日。
- 展示已经走过的珍贵回忆。
- 添加第一个事件时显示空状态引导。
- 点击事件进入详情。

### 6.3 新建事件

- 选择纪念日或倒数日。
- 输入标题。
- 选择日期。
- 标记公历或农历。
- 选择标签：自己、我们、家人、未来、其他。
- 添加备注。
- 开启或关闭提醒。
- 保存事件。

### 6.4 事件详情

- 展示距离目标日期的天数。
- 根据事件类型和标签生成文案。
- 直接编辑标题。
- 直接编辑备注。
- 修改日期。
- 开关提醒。
- 分享事件。
- 删除事件。

### 6.5 当前未完成或受限

- 农历目前只有标记，真实农历日期换算逻辑仍是占位。
- 提醒开关已保存，但没有完整的消息订阅或系统通知流程。
- 添加到手机日历的方法存在，但当前保存流程未实际调用。
- 生成纪念卡片功能仍为“开发中”。
- 纪念卡片页面为空白占位页面。
- 纪念日设置页只能清空全部纪念日，且主界面没有明显入口。

## 7. 我的厨房

### 7.1 模块作用

提供点菜、结算、做饭打卡、订单食记和简单厨房管理体验。

### 7.2 主页面四个标签

- 今日：今日菜单、做饭状态和年度热力图。
- 点菜：选择菜品和结算。
- 食记：查看历史订单。
- 厨房：厨房相关功能入口。

页面采用懒加载标签，首次切换到标签时才创建对应内容。

### 7.3 今日

- 展示当前周日期条。
- 展示今日状态：未计划、已点菜、已完成。
- 展示今日菜单。
- 跳转到点菜标签。
- 完成做饭打卡。
- 完成后写入食记。
- 展示年度做饭热力图。
- 跳转查看食记。

### 7.4 点菜

- 菜品名称搜索。
- 按分类筛选。
- 菜品数量增加和减少。
- 展示商品总数量和总价。
- 打开购物车抽屉。
- 在购物车中增加、减少或删除菜品。
- 进入结算页面。

当前菜品和分类是写在代码中的演示数据，并未与菜单管理页面联动。

### 7.5 结算

- 展示订单明细与合计金额。
- 填写地址。
- 填写备注。
- 选择微信支付或余额支付展示状态。
- 提交后记录订单。
- 云端订单写入失败时仍保留本地订单。
- 将今日状态更新为已下单。

这里不是真实支付流程，只是订单记录。

### 7.6 食记

- 展示历史订单。
- 查看订单详情。
- 将旧订单标记为完成。
- 从空状态跳回今日页面。

### 7.7 厨房标签

当前可操作：

- 查看模拟个人信息。
- 查看模拟邀请码。
- 查看默认厨房提示。
- 查看菜品分类提示。
- 添加本地临时食材。
- 查看本地临时食材。
- 联系客服、意见反馈、分享提示。

大量入口目前只展示弹窗或 Toast，不是完整业务功能。

### 7.8 菜单管理相关页面

页面文件已存在，主要使用演示数据：

- 分类新增、编辑、删除、上下排序。
- 菜品按分类展示。
- 菜品新增、编辑、删除、上下排序。
- 菜品选择图片。
- 管理分组、导入菜品仍为待开发。

这些页面当前没有从厨房主流程稳定接入，数据也没有持久化。

### 7.9 空白占位页面

- 独立购物车页面为空白占位。
- 独立订单列表页面为空白占位。

## 8. 体重管理

### 8.1 模块作用

记录每日体重，查看变化趋势和身体指标，并提供轻量搭子互动。

### 8.2 当前体重与记录

- 展示当前或所选日期的体重。
- 展示与上一次记录相比的变化。
- 根据变化生成鼓励反馈。
- 选择日期补录体重。
- 选择今日状态：正常、浮肿、吃多了、没睡好。
- 保存记录。
- 同一天只能保存一次。
- 对明显异常的体重数值进行二次确认。
- 展示最近五条记录。

### 8.3 趋势和身体指标

- 7 天与 30 天趋势图切换。
- 隐藏或显示体重数字。
- 填写身高、出生年份和性别。
- 计算 BMI。
- 估算体脂率。

### 8.4 搭子功能

- 生成并复制邀请码。
- 输入邀请码绑定搭子。
- 展示自己和搭子的今日记录状态。
- 展示连续记录天数。
- 展示搭子今日体重变化。
- 给搭子发送鼓励。

当前搭子功能为本机双用户数据模拟，不是真正的跨账号协作。

### 8.5 数据

- 体重记录采用本地 XOR 编码存储。
- 个人中心的自动同步机制会将本地记录镜像到云端。

## 9. 月经记录

### 9.1 模块作用

记录经期、症状、情绪和备注，并依据历史记录提供周期预测。

### 9.2 状态与预测

- 展示当前周期第几天。
- 预计下次日期。
- 展示距离预计日期的剩余或延迟天数。
- 展示平均周期。
- 展示平均经期长度。
- 估算排卵日期。
- 估算易孕期范围。
- 明确提示预测仅供参考，不用于诊断或避孕判断。

### 9.3 快捷记录

- 点击“今天开始”快速创建进行中的记录。
- 点击“今天结束”结束当前记录。
- 防止同时存在多个进行中记录。

### 9.4 新增与编辑历史记录

- 选择开始日期。
- 可选结束日期，或保留为进行中。
- 记录经量：少量、正常、偏多。
- 记录疼痛程度 0-10。
- 多选症状：腹痛、腰酸、头痛、乏力、胀气、长痘。
- 多选情绪：稳定、敏感、焦虑、低落。
- 添加备注。
- 编辑历史记录。
- 删除历史记录。

### 9.5 设置与隐私

- 自定义周期天数。
- 自定义经期天数。
- 开启隐私模式后隐藏日期、周期和历史详情。
- 数据在本地编码存储，并同步云端镜像。

## 10. 个人中心与云端数据

### 10.1 个人资料

- 从相册或相机选择头像。
- 修改昵称。
- 展示登录状态和会员状态。
- 展示陪伴天数、累计记录和连续打卡。

累计记录会统计纪念日、厨房订单、体重、月经和本地回本记录。

### 10.2 云端账号

- 手机号和密码注册。
- 手机号和密码登录。
- 密码为 6-32 位，不能包含空格。
- 登录或注册成功后自动恢复该账号云端数据。
- 当前账号仅显示脱敏手机号。
- 支持切换账号。
- 数据按云端账号 ID 隔离。

当前没有独立“退出登录”按钮。

### 10.3 数据管理

- 同步云端：上传本机数据和业务集合。
- 云端恢复：使用当前账号的备份覆盖本机同名数据。
- 导出备份：复制本机数据 JSON 到剪贴板。
- 清空数据：先备份，再清空本机和当前账号的业务记录。
- 展示最近同步时间、当前账号和本机记录数。
- 所有操作使用自定义确认、结果和错误弹层。

### 10.4 自动同步范围

本地存储数据写入时会自动镜像到云端，账号凭证和 OpenID 等身份字段除外。

会单独备份的云数据库集合：

- `categories`
- `goods`
- `roi_logs`
- `use_logs`
- `orders`

### 10.5 偏好与支持

- 每日提醒开关。
- 通知时间选择。
- 主题偏好：默认、清爽蓝、暖阳橙。
- 查看会员状态。
- 查看权益。
- 新手教程。
- 意见反馈说明。
- 联系客服说明。
- 关于和版本信息。

每日提醒和主题偏好当前主要完成了设置保存，尚未形成完整的全局视觉主题或通知执行流程。

## 11. UI 必须覆盖的通用状态

所有模块设计时都应考虑：

- 首次进入和空数据状态。
- 加载状态。
- 成功状态。
- 网络失败和接口失败状态。
- 表单校验错误。
- 删除和清空的危险确认。
- 无权限状态，例如定位权限未开启。
- 云端未初始化或恢复失败。
- 缓存数据状态。
- 隐私隐藏状态。
- 禁用和规划中状态。
- 页面底部安全区。
- 长文本、长商品名、长城市名和大数字。

## 12. 给 UI 设计 AI 的直接任务说明

请为一个名为“小小工具箱”的微信小程序设计完整 UI。它不是营销网站，而是高频使用的生活工具集合，设计需要安静、清晰、轻量、易扫描。

设计范围包括：首页、个人中心、到期便签与使用成本、天气预报、纪念日、我的厨房、体重管理、月经记录。

设计要求：

1. 保留各模块独立气质，但使用统一的导航、间距、字体、按钮、弹层、空状态和表单规范。
2. 首页采用工具入口布局，突出最近使用模块，不要做大型营销 Hero。
3. 数据密集页面优先保证信息层级、筛选效率和可读性，避免堆叠大量装饰卡片。
4. 天气页需要有天气与昼夜变化感，但文字在浅色、白色和复杂背景上都必须清晰。
5. 到期便签需要同时适配“到期提醒”和“使用成本”两种模式。
6. 月经记录强调隐私、安全和柔和感，但不要过度粉色化。
7. 体重管理强调趋势和长期记录，不要制造体重焦虑。
8. 厨房模块允许更温暖活泼，但点菜与结算流程必须高效。
9. 所有主要按钮文字必须水平和垂直居中。
10. 必须设计空数据、加载、失败、禁用、缓存、隐私隐藏、危险确认等状态。
11. 明确区分正式功能、演示功能、规划中功能，不要把占位入口设计成已完整可用。
12. 输出每个页面的布局结构、组件清单、交互状态、颜色与字体规范，并提供关键页面高保真方案。
